# 财务第三方支付核对 Windows Qt 程序需求说明与设计方案

## 1. 项目目标

本程序用于按店铺核对第三方支付平台资金流水。用户每月导入不同店铺的已结算或未结算流水表，系统按店铺、报表类型、年月区间汇总金额，并将系统计算出的“结算期末余额”与用户录入的“店铺期末余额”进行核对，发现差异后展示差异明细并支持手工调整留痕。

程序要求：

- Windows 桌面应用。
- Qt/PySide6 开发。
- SQLite 本地落盘。
- 支持历史查询、增删改查、导入、导出、备份还原。
- 支持打包为 exe 并发布到 GitHub Release。

## 2. 核心业务需求

### 2.1 店铺管理

- 系统支持配置多个店铺。
- 导入流水时必须选择店铺。
- 如果表格中没有店铺字段，则按导入时选择的店铺归属处理。
- 店铺名称可以直接编辑，编辑后历史汇总、明细、余额、调整记录和字段配置同步更新。
- 每个店铺使用独立 SQLite 数据库，主库只保存店铺清单和全局配置。

### 2.2 报表类型

- 导入流水时选择报表类型。
- 默认报表类型：已结算、未结算。
- 汇总、明细、余额、差异、调整和导出均按店铺与报表类型隔离。

### 2.3 年月归属

- 导入流水时选择归属年份和月份。
- 年份范围：2025 年到 2099 年。
- 默认选择当前系统年月。
- 首页查询使用年月区间，支持从某月到某月的连续区间。
- 汇总和明细界面统一显示“年月”，内部仍兼容原有“月份”字段配置。

### 2.4 汇总字段

初始汇总字段：

| 字段 | 说明 |
| --- | --- |
| 店铺 | 当前店铺 |
| 报表类型 | 已结算或未结算 |
| 年月 | 归属年月 |
| 订单实付应结 | 原始表格字段汇总 |
| 平台补贴 | 原始表格字段汇总 |
| 商家补贴 | 原始表格字段汇总 |
| 结算运费 | 原始表格字段汇总 |
| 订单退款 | 原始表格字段汇总 |
| 收入净额合计 | 订单实付应结 + 平台补贴 + 商家补贴 + 结算运费 + 订单退款 |
| 已结算佣金 | 原始佣金字段汇总 |
| 技术服务费 | 原始技术服务费字段汇总 |
| 支出金额 | 已结算佣金 + 技术服务费 |
| 结算金额 | 收入净额合计 + 支出金额 |
| 提现金额 | 提现流水汇总 |
| 期初金额 | 用户录入 |
| 结算期末余额 | 期初金额 + 收入净额合计 + 支出金额 + 提现金额 |
| 店铺期末余额 | 用户录入 |
| 差异 | 结算期末余额 - 店铺期末余额 |
| 差异原因 | 系统生成说明 + 手工调整说明 |

### 2.5 余额核对

- 用户可录入期初金额和店铺期末余额。
- 系统根据流水汇总计算结算期末余额。
- 差异 = 结算期末余额 - 店铺期末余额。
- 多月区间汇总时：
  - 期初金额取区间首月期初。
  - 店铺期末余额取区间末月期末。
  - 期初和期末不做累加。
  - 明细流水按区间汇总显示。

### 2.6 差异处理

- 当结算期末余额与店铺期末余额不一致时，系统显示差异金额和差异说明。
- 差异明细页面使用表格展示核对摘要、动账分组和手工调整记录。
- 用户可以新增手工调整：
  - 选择调整影响的汇总列。
  - 录入调整金额。
  - 录入调整说明。
  - 调整动作必须落库留痕。
- 调整说明在汇总表中展示。

### 2.7 明细流水

- 明细流水默认显示当前筛选年月或年月区间的数据。
- 默认按动账时间升序排序。
- 点击表头可按任意字段升序/降序排序。
- 支持分页，默认每页 1000 行。
- 支持跳到首页、上一页、下一页、末页和指定页。
- 支持多字段精确筛选：
  - 先选择字段和值。
  - 点击“加入添加筛选条件”。
  - 可连续加入多个条件。
  - 点击“开始筛选”后再查询大表。
  - 点击“清除筛选条件”清空筛选。
- 筛选条件过多时自动换行显示。

### 2.8 参数配置

每个店铺可单独维护参数配置：

- 原始表格字段。
- 汇总表格字段。
- 冻结栏字段。
- 活动栏字段。
- 字段顺序。
- 计算口径。
- 分页行数，默认 1000。

导入某店铺表格时，系统可根据表头自动初始化原始字段配置，后续可手工调整。

### 2.9 导入导出

- 支持导入 `.xlsx`、`.xlsm`、`.csv`。
- CSV 兼容 UTF-8 BOM 和 GB18030。
- 导入时清理文本前导标记，例如订单号、流水号前面的 `'`。
- 导入时显示真实处理行数和进度。
- 导出 XLSX 包含：
  - 汇总表。
  - 原始表格。
  - 店铺名称。
  - 报表类型。
  - 年月区间。
  - 导出时间戳。
- 导出按分页配置分批读取并流式写入，避免大数据一次性加载卡顿。
- 导出完成弹窗提供“打开文件”按钮。

### 2.10 备份还原

- 支持备份全部店铺或部分店铺。
- 支持还原全部店铺或部分店铺。
- 备份内容包括：
  - 主库店铺配置。
  - 全局设置。
  - 店铺独立 SQLite 数据库。
- 备份文件为 ZIP。

### 2.11 全局设置

- 支持开机自启动。
- 支持点击窗口 X 时最小化到托盘而不是退出。
- 设置保存到主库。

## 3. 技术方案

### 3.1 技术栈

- 语言：Python 3.12
- UI 框架：PySide6 / Qt
- 数据库：SQLite
- Excel 处理：openpyxl
- 打包：PyInstaller
- 发布：GitHub Repository + GitHub Release

### 3.2 数据库设计

主库：

```text
data/reconciliation.sqlite3
```

职责：

- 店铺清单。
- 店铺数据库文件名。
- 全局设置。

店铺库：

```text
data/stores/{store_name}_{hash}.sqlite3
```

职责：

- 原始流水。
- 余额记录。
- 手工调整。
- 原始字段配置。
- 汇总字段配置。
- 冻结栏字段。
- 分页配置。

### 3.3 数据隔离设计

将不同店铺数据拆到不同 SQLite 文件中，目的：

- 降低单库数据量。
- 减少切换店铺时的查询范围。
- 便于单店铺备份和还原。
- 避免不同店铺字段配置相互影响。

### 3.4 UI 设计

首页结构：

- 顶部工具栏：导入流水、店铺、参数、余额、调整、差异、导出、备份、设置。
- 筛选区：店铺、报表类型、年月区间。
- 汇总区：左侧冻结栏，右侧活动栏，中间可拖动调宽。
- 明细区：明细表格与底部控制区使用纵向 splitter，可拖动调整高度。
- 底部控制区：筛选字段、筛选值、筛选按钮、分页按钮、编辑/删除按钮、状态信息。
- 状态提示：单行胶囊样式，长文本使用 tooltip。

### 3.5 性能优化

- 大表导入时使用进度回调，避免用户误以为程序卡死。
- 切换店铺时先清空界面，不立即加载大数据。
- 查询明细时默认分页加载。
- 筛选条件先暂存，点击“开始筛选”后再查询。
- 筛选值使用缓存，避免重复读取 distinct 值。
- 导出原始流水按分页配置分批读取。
- 表格关闭自动换行，减少大表渲染高度计算。
- 对常用查询字段建立索引。
