# 财务第三方支付核对 Qt 版

一个用于店铺第三方支付资金流水核对的 Windows 桌面应用，使用 Qt/PySide6 开发，数据使用 SQLite 本地落盘。

## 版本

当前版本：`v1.0.2`

## 功能

- 店铺配置：先维护店铺，导入 Excel 时选择本次归属店铺。
- Excel 导入：支持资金已结算流水明细，按店铺和月份汇总。
- 无店铺表头处理：如果 Excel 没有 `店铺` 表头，会按导入时选择的店铺处理。
- 余额核对：录入期初金额和店铺期末余额，系统计算结算期末余额并核对差异。
- 差异明细：按动账方向/摘要展示差异相关汇总，并保留差异原因。
- 手工调整：可选择调整到指定汇总列，调整动作会被记录；选择“备查（不影响汇总）”只记录说明。
- 调整说明：手工调整说明会显示在汇总表中。
- 多月份筛选：支持月份多选和区间输入，例如 `202606-202607`，汇总表会显示区间合计和单月行。
- 区间余额：多月份汇总时，期初金额取首月期初，店铺期末余额取末月期末，期初/期末不累加。
- 明细流水：默认显示筛选月份或区间内的流水，按动账时间升序排序，可切换降序；点击单月行显示单月明细，点击区间合计行显示区间明细。
- 店铺配置：可维护每个店铺的原始表格字段、汇总表格字段和计算口径说明；导入店铺表格时会自动生成/补充原始字段。
- 抖音/平台字段自动识别：支持 `动账/动帐` 字段别名，缺少摘要时会使用 `动账场景/计费类型` 初始化。
- 历史维护：支持查询、导出 CSV、编辑/删除流水、删除选中月份流水。
- 表格体验：汇总表冻结 `店铺`、`月份` 两列，后续列可横向拖动查看。

## 计算口径

```text
收入净额合计 = 订单实付应结 + 平台补贴 + 商家补贴 + 结算运费 + 订单退款
支出金额 = 已结算佣金 + 技术服务费
结算金额 = 收入净额合计 + 支出金额
结算期末余额 = 期初金额 + 收入净额合计 + 支出金额 + 提现金额
差异 = 结算期末余额 - 店铺期末余额
```

## 汇总表字段

店铺、月份、订单实付应结、平台补贴、商家补贴、结算运费、订单退款、收入净额合计、已结算佣金、技术服务费、支出金额、结算金额、提现金额、期初金额、结算期末余额、店铺期末余额、差异、差异原因。

## 数据存储

SQLite 数据库保存在程序目录下：

```text
data/reconciliation.sqlite3
```

数据库文件不会提交到源码仓库，程序首次运行时会自动创建。

## 开发运行

需要 Python 3.12，安装依赖：

```powershell
python -m pip install -r requirements.txt
```

启动：

```powershell
python qt_app.py
```

也可以双击 `启动Qt版.bat`。

## 打包

使用 PyInstaller 打包：

```powershell
python -m PyInstaller --noconfirm --clean --windowed --onefile --name "财务第三方支付核对-Qt版" qt_app.py
```

打包完成后，exe 位于 `dist/财务第三方支付核对-Qt版.exe`。
